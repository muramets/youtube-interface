# Руководство по запуску приложения (для начинающих)

Этот файл поможет вам шаг за шагом запустить приложение и Storybook.

## 1. Подготовка (Терминал)

Все команды выполняются в **Терминале**.

1.  Откройте программу **Terminal** (или iTerm, VS Code Terminal).
2.  Вам нужно перейти в папку с проектом. Введите команду `cd` (change directory), пробел и путь к папке.
    
    Пример (если проект в Документах):
    ```bash
    cd /Users/muramets/Documents/youtube-interface
    ```
    *Совет: Вы можете написать `cd ` (с пробелом) и просто перетащить папку проекта из Finder в окно терминала, путь подставится сам.*

## 2. Установка зависимостей

Перед первым запуском нужно скачать все необходимые библиотеки ("детали" для сборки приложения). Это делается один раз (или если вы добавили новые библиотеки).

Введите команду:
```bash
npm install
```
*Что она делает:* Читает файл `package.json` и скачивает все нужные модули в папку `node_modules`. Это может занять пару минут.

## 3. Запуск приложения (Режим разработки)

Чтобы увидеть приложение в браузере и работать над ним:

Введите команду:
```bash
npm run dev
```
*Что она делает:* Запускает локальный сервер разработки.
*Результат:* В терминале появится ссылка, обычно `http://localhost:5173`. Нажмите на неё (обычно Cmd + Click) или скопируйте в браузер.
*Особенности:* В этом режиме изменения в коде сразу обновляют страницу в браузере (Hot Reload).

## 4. Запуск Storybook

Storybook — это "каталог" всех компонентов (кнопок, карточек и т.д.), где их можно посмотреть отдельно от остального сайта.

Введите команду:
```bash
npm run storybook
```
*Что она делает:* Запускает отдельный локальный сервер для Storybook.
*Результат:* Обычно открывается автоматически по адресу `http://localhost:6006`.

## 5. Сборка проекта (Build)

Если вы хотите получить готовую версию сайта для выкладки в интернет (Production):

Введите команду:
```bash
npm run build
```
*Что она делает:* Проверяет код на ошибки и "собирает" оптимизированные файлы в папку `dist`.
*Зачем:* Эти файлы весят меньше и работают быстрее, но их нельзя легко редактировать.

## Краткая шпаргалка

| Команда | Что делает | Когда использовать |
| :--- | :--- | :--- |
| `cd [путь]` | Переход в папку | В самом начале |
| `npm install` | Установка библиотек | Первый раз или после обновлений |
| `npm run dev` | Запуск сайта | Для работы и просмотра |
| `npm run storybook` | Запуск каталога компонентов | Для проверки верстки компонентов |
| `Control + C` | Остановка сервера | Чтобы выключить dev или storybook |
